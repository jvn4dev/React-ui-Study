## 커밋 메세지 가이드

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포매팅, 코드 변경 없이 스타일 관련 수정사항
- `refactor`: 코드 리팩토링(기능변경없는)
- `test`: 테스트 코드,프로덕션 코드 수정 없는
- `chore`: 빌드 업무, 설정, 기타 변경사항

> 예시 :

```
feat: 상세 페이지 ThumnailList에 무한 스크롤 기능 추가
chore: 폴더 구조 변경
docs: dropdown-readme 문서 작성
```
